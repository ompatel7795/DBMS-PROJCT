Section A: Supplier Table (Questions 1-10)

Table Structure:

CREATE TABLE Supplier (
    Sup_No VARCHAR(10) PRIMARY KEY,
    Sup_Name VARCHAR(50),
    Item_Supplied VARCHAR(50),
    Item_Price INT,
    City VARCHAR(50)
);
1

Display Supplier numbers and names whose name starts with 'R'

SELECT Sup_No, Sup_Name 
FROM Supplier 
WHERE Sup_Name LIKE 'R%';
Explanation: Uses LIKE operator with wildcard 'R%' to match names starting with 'R'. The percent sign (%) matches any sequence of characters after 'R'.
2

Display names of suppliers who supply Processors and city is Delhi

SELECT Sup_Name 
FROM Supplier 
WHERE Item_Supplied = 'Processor' AND City = 'Delhi';
Explanation: Uses AND operator to combine two conditions: Item_Supplied must be 'Processor' AND City must be 'Delhi'. Only records meeting both conditions are returned.
3

Display names of suppliers who supply same items as supplied by Ramesh

SELECT Sup_Name 
FROM Supplier 
WHERE Item_Supplied IN (
    SELECT Item_Supplied 
    FROM Supplier 
    WHERE Sup_Name = 'Ramesh'
);
Explanation: Uses a subquery to first find what items Ramesh supplies, then finds all suppliers who supply those same items. The IN operator checks if Item_Supplied matches any value from the subquery result.
4

Increase the price of Keyboard by 200

UPDATE Supplier 
SET Item_Price = Item_Price + 200 
WHERE Item_Supplied = 'Keyboard';
Explanation: UPDATE statement modifies existing records. SET clause adds 200 to current Item_Price value. WHERE clause ensures only Keyboard items are updated.
5

Display supplier numbers, names and itemprice for Delhi suppliers in ascending order of itemprice

SELECT Sup_No, Sup_Name, Item_Price 
FROM Supplier 
WHERE City = 'Delhi' 
ORDER BY Item_Price ASC;
Explanation: Filters records where City is 'Delhi', then sorts results by Item_Price in ascending order (lowest to highest). ASC is optional as it's the default sort order.
6

Add a new column called CONTACTNO

ALTER TABLE Supplier 
ADD COLUMN CONTACTNO VARCHAR(15);
Explanation: ALTER TABLE modifies table structure. ADD COLUMN adds a new column named CONTACTNO with VARCHAR(15) data type to store phone numbers.
7

Delete the record whose itemprice is the lowest

DELETE FROM Supplier 
WHERE Item_Price = (
    SELECT MIN(Item_Price) 
    FROM Supplier
);
Explanation: Subquery finds the minimum Item_Price using MIN() function. DELETE statement removes records where Item_Price equals this minimum value.
8

Create a view displaying only supplier numbers and names

CREATE VIEW SupplierView AS 
SELECT Sup_No, Sup_Name 
FROM Supplier;
Explanation: CREATE VIEW creates a virtual table that stores a query. This view will always show current Sup_No and Sup_Name data from the Supplier table without storing duplicate data.
9

Display records in descending order of itemprice for each itemsupplied

SELECT * 
FROM Supplier 
ORDER BY Item_Supplied, Item_Price DESC;
Explanation: ORDER BY with multiple columns: first groups by Item_Supplied, then within each item group, sorts by Item_Price in descending order (highest to lowest).
10

Display suppliers who supply items other than Processor or Keyboard

SELECT * 
FROM Supplier 
WHERE Item_Supplied NOT IN ('Processor', 'Keyboard');
Explanation: NOT IN operator excludes records where Item_Supplied matches any value in the list. This returns all suppliers who supply items other than Processor or Keyboard.

Section B: EmpDetails Table (Questions 11-20)

Table Structure:

CREATE TABLE EmpDetails (
    Eid VARCHAR(10) PRIMARY KEY,
    Ename VARCHAR(50),
    DOB DATE,
    Designation VARCHAR(50),
    Salary INT,
    DOJ DATE
);
11

Display all employees whose designation is Programmer

SELECT * 
FROM EmpDetails 
WHERE Designation = 'Programmer';
Explanation: Simple WHERE clause filters records to show only employees with Designation equal to 'Programmer'. Returns all columns for matching records.
12

Display employees who have joined after 2014

SELECT * 
FROM EmpDetails 
WHERE YEAR(DOJ) > 2014;
Explanation: YEAR() function extracts the year from DOJ (Date of Joining). Comparison operator > filters employees who joined after year 2014.
13

Display all employees whose name ends with 'a'

SELECT * 
FROM EmpDetails 
WHERE Ename LIKE '%a';
Explanation: LIKE operator with '%a' pattern matches names ending with 'a'. The percent sign matches any characters before the final 'a'.
14

Display total salary of all employees whose designation is Programmer

SELECT SUM(Salary) AS TotalSalary 
FROM EmpDetails 
WHERE Designation = 'Programmer';
Explanation: SUM() aggregate function adds up all Salary values for employees with designation 'Programmer'. AS keyword creates an alias 'TotalSalary' for the result column.
15

Display all employee names in upper case

SELECT UPPER(Ename) AS EmployeeName 
FROM EmpDetails;
Explanation: UPPER() string function converts all characters in Ename to uppercase. Returns transformed names without modifying the actual database data.
16

Display details of employee with highest experience

SELECT * 
FROM EmpDetails 
WHERE DOJ = (
    SELECT MIN(DOJ) 
    FROM EmpDetails
);
Explanation: Highest experience means earliest joining date. MIN(DOJ) finds the minimum (earliest) date of joining. Main query selects employee(s) with that earliest DOJ.
17

Display employees whose name contains 'ee'

SELECT * 
FROM EmpDetails 
WHERE Ename LIKE '%ee%';
Explanation: LIKE with '%ee%' pattern matches names containing 'ee' anywhere in the string. Both percent signs allow any characters before and after 'ee'.
18

Increase salaries by 5000 for employees whose designation is DBA

UPDATE EmpDetails 
SET Salary = Salary + 5000 
WHERE Designation = 'DBA';
Explanation: UPDATE statement modifies existing records. Adds 5000 to current Salary for all employees with designation 'DBA'. WHERE clause ensures only DBAs are affected.
19

Display employees whose salary is more than average salary

SELECT * 
FROM EmpDetails 
WHERE Salary > (
    SELECT AVG(Salary) 
    FROM EmpDetails
);
Explanation: Subquery calculates average salary using AVG() function. Main query compares each employee's salary against this average and returns those earning more.
20

Display in format: "Name is working as Designation with a Salary of Rs.Amount"

SELECT CONCAT(Ename, ' is working as ', Designation, ' with a Salary of Rs.', Salary) AS EmployeeInfo 
FROM EmpDetails;
Explanation: CONCAT() function joins multiple strings and column values into one formatted string. Creates a custom message format for each employee with their name, designation, and salary.

Section C: Employee-Department Tables (Questions 21-30)

Table Structures:

CREATE TABLE Department (
    DeptId VARCHAR(10) PRIMARY KEY,
    Dname VARCHAR(50) NOT NULL
);

CREATE TABLE Employee (
    Eid INT PRIMARY KEY,
    Ename VARCHAR(50),
    DeptId VARCHAR(10),
    Designation VARCHAR(50),
    Salary INT CHECK (Salary > 10000),
    DOJ DATE,
    FOREIGN KEY (DeptId) REFERENCES Department(DeptId)
);
21

Display employees who earn more than average salary

SELECT * 
FROM Employee 
WHERE Salary > (
    SELECT AVG(Salary) 
    FROM Employee
);
Explanation: Subquery calculates company-wide average salary. Main query filters employees whose salary exceeds this average, showing above-average earners.
22

Display Eid, Ename and Dname

SELECT E.Eid, E.Ename, D.Dname 
FROM Employee E 
INNER JOIN Department D ON E.DeptId = D.DeptId;
Explanation: INNER JOIN combines Employee and Department tables based on matching DeptId values. Returns employee ID, name from Employee table and department name from Department table.
23

Sort employee table in descending order of salaries

SELECT * 
FROM Employee 
ORDER BY Salary DESC;
Explanation: ORDER BY with DESC sorts records from highest to lowest salary. Shows employees ranked by salary with highest earners first.
24

List all job designations without repetitions

SELECT DISTINCT Designation 
FROM Employee;
Explanation: DISTINCT keyword removes duplicate values from result set. Returns unique list of all job designations that exist in the company.
25

Display employee details Department wise in ascending order of salaries

SELECT * 
FROM Employee 
ORDER BY DeptId, Salary ASC;
Explanation: Multi-level sorting: first groups by DeptId, then within each department sorts by Salary in ascending order (lowest to highest).
26

Display all clerks in DeptId D2

SELECT * 
FROM Employee 
WHERE Designation = 'Clerk' AND DeptId = 'D2';
Explanation: AND operator combines two conditions: Designation must be 'Clerk' AND DeptId must be 'D2'. Returns only clerks working in department D2.
27

Display employees who joined in year 2011

SELECT * 
FROM Employee 
WHERE YEAR(DOJ) = 2011;
Explanation: YEAR() function extracts year from DOJ date field. Filters employees whose joining year equals 2011.
28

Display employees who joined in month of February

SELECT * 
FROM Employee 
WHERE MONTH(DOJ) = 2;
Explanation: MONTH() function extracts month number from DOJ date (1=January, 2=February, etc.). Filters employees who joined in February (month 2).
29

Display employees whose salary is between 30000 and 45000

SELECT * 
FROM Employee 
WHERE Salary BETWEEN 30000 AND 45000;
Explanation: BETWEEN operator checks if salary falls within specified range (inclusive of both 30000 and 45000). More readable than using >= and <= operators.
30

Display employee details with work experience till current date

SELECT *, 
    TIMESTAMPDIFF(YEAR, DOJ, CURDATE()) AS Experience_Years 
FROM Employee;
Explanation: TIMESTAMPDIFF() calculates difference between DOJ and current date (CURDATE()) in years. Shows all employee details plus their years of experience.

Section D: Student Table (Questions 31-40)

Table Structure:

CREATE TABLE Student (
    Sid INT PRIMARY KEY,
    Sname VARCHAR(50),
    DOB DATE,
    State VARCHAR(50),
    Gender CHAR(1),
    Category VARCHAR(50),
    Course VARCHAR(50)
);
31

Display students not from Telangana or AndhraPradesh

SELECT * 
FROM Student 
WHERE State NOT IN ('Telangana', 'AndhraPradesh');
Explanation: NOT IN operator excludes students from specified states. Returns students from all states except Telangana and AndhraPradesh.
32

Create view displaying Sid, Sname for students from Telangana

CREATE VIEW TelanganaStudents AS 
SELECT Sid, Sname 
FROM Student 
WHERE State = 'Telangana';
Explanation: CREATE VIEW creates a virtual table showing only Telangana students' IDs and names. View automatically updates when base table changes.
33

Create index on column Sname

CREATE INDEX idx_Sname ON Student(Sname);
Explanation: CREATE INDEX creates an index on Sname column to speed up search queries. Improves performance when filtering or sorting by student name.
34

Display female students enrolled in Comp course who belong to OBC

SELECT * 
FROM Student 
WHERE Gender = 'F' AND Course = 'Comp' AND Category = 'OBC';
Explanation: Multiple AND conditions filter students who meet all three criteria: Female gender, Comp course, and OBC category.
35

Display student IDs, names and present age

SELECT Sid, Sname, 
    TIMESTAMPDIFF(YEAR, DOB, CURDATE()) AS Age 
FROM Student;
Explanation: TIMESTAMPDIFF() calculates age by finding years between date of birth (DOB) and current date (CURDATE()). Returns student ID, name and calculated age.
36

Display students in ascending order of names for each course

SELECT * 
FROM Student 
ORDER BY Course, Sname ASC;
Explanation: Multi-level sorting: first groups by Course, then within each course sorts students alphabetically by name in ascending order.
37

Delete students enrolled in Comp course who were born after 2002

DELETE FROM Student 
WHERE Course = 'Comp' AND YEAR(DOB) > 2002;
Explanation: DELETE removes records matching both conditions: Course is 'Comp' AND birth year is after 2002. YEAR() extracts year from DOB date.
38

Add two new columns Contactno and Email

ALTER TABLE Student 
ADD COLUMN Contactno VARCHAR(15), 
ADD COLUMN Email VARCHAR(100);
Explanation: ALTER TABLE with multiple ADD COLUMN statements adds two new columns to existing table structure. Both columns start with NULL values for existing records.
39

Display student names prefixed with Mr./Ms. based on gender

SELECT 
    CASE 
        WHEN Gender = 'M' THEN CONCAT('Mr. ', Sname)
        WHEN Gender = 'F' THEN CONCAT('Ms. ', Sname)
    END AS StudentName 
FROM Student;
Explanation: CASE expression checks Gender value. If Male, prefixes 'Mr.'; if Female, prefixes 'Ms.'. CONCAT() joins prefix with student name.
40

Display student names where length is 5 characters

SELECT Sname 
FROM Student 
WHERE LENGTH(Sname) = 5;
Explanation: LENGTH() function returns number of characters in string. Filters students whose name has exactly 5 characters.

Section E: Library Table (Questions 41-50)

Table Structure:

CREATE TABLE Library (
    BookId VARCHAR(10) PRIMARY KEY,
    BookName VARCHAR(100) NOT NULL,
    Author VARCHAR(50),
    DatePurchased DATE,
    Publisher VARCHAR(50),
    Price INT
);
41

Display list of authors from Himalaya publications

SELECT DISTINCT Author 
FROM Library 
WHERE Publisher = 'Himalaya';
Explanation: Filters books from Himalaya publisher and returns unique author names using DISTINCT to eliminate duplicates if same author has multiple books.
42

Display total cost of books purchased Publisher wise

SELECT Publisher, SUM(Price) AS TotalCost 
FROM Library 
GROUP BY Publisher;
Explanation: GROUP BY groups books by Publisher. SUM(Price) calculates total cost for each publisher group. Shows spending per publisher.
43

Count total number of books under Kalyani publications

SELECT COUNT(*) AS TotalBooks 
FROM Library 
WHERE Publisher = 'Kalyani';
Explanation: COUNT(*) counts number of records. WHERE clause filters only Kalyani publisher books. Returns total count of Kalyani books.
44

Rename column Publisher to Publications

ALTER TABLE Library 
CHANGE COLUMN Publisher Publications VARCHAR(50);
Explanation: ALTER TABLE with CHANGE COLUMN renames Publisher column to Publications. Must specify data type (VARCHAR(50)) even when not changing it.
45

Display books in ascending order of DatePurchased

SELECT * 
FROM Library 
ORDER BY DatePurchased ASC;
Explanation: ORDER BY sorts books by DatePurchased in ascending order (oldest to newest). Shows books in chronological purchase order.
46

Create index on fields BookName and Author

CREATE INDEX idx_BookName_Author ON Library(BookName, Author);
Explanation: Creates composite index on both BookName and Author columns. Improves performance for queries that search or sort by book name and/or author.
47

Display books whose price is between 500 and 700

SELECT * 
FROM Library 
WHERE Price BETWEEN 500 AND 700;
Explanation: BETWEEN operator filters books with price in range 500 to 700 (inclusive). Returns books in this price range.
48

Increase price by 200 for publishers other than Himalaya or Kalyani

UPDATE Library 
SET Price = Price + 200 
WHERE Publisher NOT IN ('Himalaya', 'Kalyani');
Explanation: UPDATE increases price by 200. NOT IN excludes Himalaya and Kalyani publishers. Only books from other publishers get price increase.
49

Display books where author name contains 'Sharma'

SELECT * 
FROM Library 
WHERE Author LIKE '%Sharma%';
Explanation: LIKE with '%Sharma%' pattern matches author names containing 'Sharma' anywhere in the string. Finds all Sharma authors regardless of first name.
50

Create view displaying BookId and BookName for Himalaya publisher

CREATE VIEW HimalayaBooks AS 
SELECT BookId, BookName 
FROM Library 
WHERE Publisher = 'Himalaya';
Explanation: CREATE VIEW creates virtual table showing only BookId and BookName for Himalaya publisher books. View updates automatically when base table changes.
